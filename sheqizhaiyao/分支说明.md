# sheqizhaiyao 分支说明

## 分支用途

本分支用于实现"社区摘要"功能：对长篇小说，按图谱社区聚类用 LLM 生成叙事摘要，向量化后用于回答全局性问题（如角色阵营、主要关系网络等）。

## 背景

当前图谱的 `CommunityInfo` 只有 `topNodes`（节点标签列表），没有 LLM 生成的语义摘要。`ContextPack.graphSearchResults` 只检索节点级关联，无法回答"主要阵营"、"角色关系网络"等全局问题。

GraphRAG 的社区摘要可以填补这一空白：对 Louvain 算法检测出的每个社区，用 LLM 生成 200-400 字叙事摘要，向量化后注入检索流程。

## 设计方案

### 核心决策

| 维度 | 决策 |
|---|---|
| 生成时机 | 定期重建（每 N 章触发，N 由用户配置，默认 5） |
| 摘要内容 | 叙事摘要（200-400 字） |
| 检索方式 | 向量检索（复用 `fetchEmbedding` + `searchByEmbedding`） |
| 重排模型 | 不使用（rerank 默认关闭，不影响功能） |
| 执行方式 | 用户可选：后台异步（默认）或同步等待 |
| 失败处理 | 弹窗提示（中文），不影响章节摄取主流程 |

### 配置项（NovelConfig 新增 3 字段）

```typescript
communitySummaryEnabled: boolean    // 默认 true，是否开启社区摘要自动提取
communitySummaryInterval: number    // 默认 5, min=1, max=50，每 N 章触发
communitySummaryAsync: boolean      // 默认 true，true=后台异步，false=同步等待
```

### 核心逻辑（新建 src/lib/novel/community-summary.ts）

**`generateCommunitySummaries(projectPath, llmConfig, novelConfig)`**：
1. 调用 `buildWikiGraph` 获取 `communities`
2. 对每个社区，收集成员节点的 wiki 内容（前 500 字/节点，最多 10 个节点）
3. 调用 `streamChat`（用 `summaryModel`）生成 200-400 字叙事摘要
4. 持久化到 `${pp}/.novel/community-summaries/{communityId}.json`
5. 向量化写入 LanceDB（`page_id = community:xxx`）

**`shouldRebuildCommunitySummaries(currentChapterNumber, novelConfig)`**：
- 返回 `communitySummaryEnabled && currentChapterNumber > 0 && currentChapterNumber % interval === 0`

### 触发时机（chapter-ingest.ts 修改）

- 在 `ingestChapter` 末尾，检查 `shouldRebuildCommunitySummaries`
- 若 `communitySummaryAsync = true`：后台异步执行（不 await，catch 错误后弹窗）
- 若 `communitySummaryAsync = false`：同步 await，失败时弹窗提示

### 检索注入（context-engine.ts 修改）

- 在 `searchGraphRelevantContent` 中，追加 `searchByEmbedding` 检索社区摘要
- 过滤 `page_id` 前缀为 `community:` 的结果
- 注入到 `graphSearchResults` 字段，标注【社区摘要】前缀

### i18n 文案（中文）

- `communitySummaryEnabled`: "社区摘要自动提取"
- `communitySummaryEnabledHint`: "开启后，每 N 章自动用 LLM 为图谱社区生成叙事摘要，用于回答全局性问题（如角色阵营、主要关系网络等）。需要先启用向量模型。"
- `communitySummaryInterval`: "社区摘要提取间隔"
- `communitySummaryIntervalHint`: "每摄取多少章后自动重建一次社区摘要。数值越小，摘要越新鲜，但 LLM 调用越频繁。"
- `communitySummaryAsync`: "后台异步执行"
- `communitySummaryAsyncHint`: "开启后，社区摘要在后台生成，不阻塞章节摄取。关闭则同步等待生成完成。"
